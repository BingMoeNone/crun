搜索高亮 — 两个粒度方案

用户输入 "model" 搜索，匹配到 --model 参数时的高亮效果

方案 A：行级高亮（推荐）

匹配的行整体用高亮样式标记，不区分具体匹配位置

搜索: model▌ (3 项)  Esc 退出搜索
──────────────────────────────────────
 ○ --model  当前会话使用的模型 [单选]
 ○ --fallback-model  默认模型过载时回退 [输入]
 ○ --teammate-mode  设置 agent 显示方式 [单选]
  已选 0 项

优点

  • 实现简单，仅改样式类
  • prompt_toolkit 原生支持
  • 不破坏原有渲染逻辑

缺点

  • 不精确指示匹配位置
方案 B：字符级高亮

匹配的具体字符用不同颜色标记出来

搜索: model▌ (3 项)  Esc 退出搜索
──────────────────────────────────────
 ○ --model  当前会话使用的模型 [单选]
 ○ --fallback-model  默认模型过载时回退 [输入]
 ○ --teammate-mode  设置 agent 显示方式 [单选]
  已选 0 项

优点

  • 精确展示匹配位置
  • 视觉反馈更丰富

缺点

  • 需要拆分每行为多个 (style, text) 片段
  • 改造 _render() 渲染管线
  • 中文分词边界可能不完美

方案 B 需要把每行拆成多段带样式的片段，复杂度明显高于 A。但视觉效果好很多。你选哪个？
